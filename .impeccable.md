## Design Context

### Users
Atlas is used by drone operators and developers validating PX4-based drone workflows, currently through SITL and eventually with companion-computer agents. Users need to monitor fleet health, telemetry freshness, command delivery, and mission execution while keeping PX4 as the flight-control authority.

### Brand Personality
Operational, calm, precise. The interface should feel like a dependable control surface: low drama, clear status hierarchy, and enough density for repeated use during flight testing.

### Aesthetic Direction
Light, utilitarian operations dashboard using the existing Atlas palette: ink, mist, field, signal, sky, and panel. Avoid marketing layouts, decorative visuals, neon control-room styling, and anything that makes safety-critical state harder to scan.

### Design Principles
- Make current vehicle state and workflow blockers immediately visible.
- Prefer compact lifecycle displays over explanatory copy.
- Use color as status language, not decoration.
- Keep mission and command surfaces visually related but distinct.
- Preserve responsive readability without hiding critical operational state.
